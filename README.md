# 맛.zip

> 개인 취향과 상황을 고려한 대화형 AI 맛집 추천 서비스
> 팀 도리토스 — 박민규, 김태우, 이건희, 이동휘

---

## 개요

**맛.zip**은 자연어 대화만으로 사용자의 상황(날씨, 시간대, 일행, 목적 등)과 선호를 반영해 즉시 맞춤형 맛집을 추천하고, 여러 리뷰를 간결하게 요약해 주는 웹/모바일 서비스 프로토타입입니다.
대화형 UI와 AI 요약, 실시간 질의응답 기능으로 기존의 복잡한 필터 기반 탐색을 대체하는 것을 목표로 합니다.

## 주요 기능

* **자연어 대화형 추천** — "비 오는 날 국물", "애견 동반 브런치"처럼 일상 언어로 질문하면 의도 파악 후 추천
* **리뷰 요약** — 수십 개의 리뷰를 3–4줄로 요약하여 추천 메뉴, 매장 특징, 방문 팁 제공
* **실시간 질의응답** — "주차 가능?", "예약 필요?" 등 즉석에서 답변 제공
* **상황별 맞춤 추천** — 기상, 시간대, 동행자 등을 고려한 추천
* **지도 시각화** — 카카오맵 API 연동, 추천 결과를 지도에 마커로 표시
* **개인화 학습** — 사용자 취향을 학습하여 추천 정확도 향상

---

## 차별점

* 대화형 UI로 복잡한 조건 입력 불필요
* AI가 리뷰를 요약해 시간 절약 제공
* 개인 취향 기반 맞춤 추천으로 반복 사용 시 정확도 향상

---

## 사용된 데이터 & 외부 연동

* **카카오맵 장소 검색 데이터** — 카카오 지도 API
* **네이버 플레이스 리뷰 데이터** — 네이버 Open API 또는 크롤링 (프로토타입용)
* **기상청 단기예보** — 공공데이터포털 (기상청)

> 실제 서비스화 시에는 각 API의 이용 약관 및 데이터 사용 정책을 엄격히 준수해야 합니다.

---

## 주요 기술 스택

* **LLM**: GPT-5.1 또는 Sonnet 4.5
* **임베딩**: Sentence-BER
* **감성분석**: KoBERT 기반 모델
* **추천알고리즘**: Collaborative Filtering + Content-based Hybrid
* **Vector Search**: 벡터 임베딩 + 유사도 검색
* **프론트엔드**: React Native Expo + TypeScript
* **지도/위치**: Kakao Maps JavaScript SDK
* **데이터 파이프라인**: 크롤링 → 전처리 → 임베딩 저장

## 아키텍처 (요약)

1. 사용자 질의 → 프론트엔드
2. LLM: 의도/엔티티 추출 → 조건 생성
3. Vector Search + DB 조회 → 후보 매장 수집
4. 감성분석 & 리뷰 요약 → 요약문 생성
5. 결과 응답 + 지도 시각화
